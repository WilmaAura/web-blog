1. Enterprise Architecture (EA)

Pengertian

Enterprise Architecture (EA) adalah kerangka/blueprint yang menggambarkan bagaimana bisnis, data, aplikasi, dan teknologi saling terhubung untuk mendukung strategi organisasi jangka panjang. EA bukan flowchart, tetapi dapat menggunakan flowchart sebagai bagian kecilnya.

Mengapa EA Dibutuhkan

  • Kompleksitas sistem IT semakin tinggi dan sulit dikelola
  • Sistem IT tidak lagi selaras dengan kebutuhan bisnis
  • Biaya maintenance mahal dan tidak efisien

Manfaat EA

  • Mengurangi biaya IT
  • Meningkatkan produktivitas
  • Menghindari sistem yang saling tidak kompatibel
  • Mendukung keputusan strategis manajemen
  • Mendukung sustainability jangka panjang

Komponen EA

  1. Business Architecture: proses dan tujuan bisnis
  2. Application Architecture: struktur dan interaksi aplikasi
  3. Data Architecture: pengelolaan dan alur data
  4. Technology / IT Architecture: server, storage, network, OS

2. Data Governence

Pengertian

Data Governance adalah seperangkat aturan, kebijakan, dan tanggung jawab untuk mengelola data agar:

  • Akurat
  • Aman
  • Konsisten
  • Dapat Dipercaya

Mengapa Data Governence Penting

  • Data adalah aset paling berharga
  • Data lintas departemen
  • Regulasi dan kepatuhan (compliance)
  • Keputusan bisnis bergantung pada kualitas data

Risiko Jika Data Tidak Dikelola

  • Data silo dan tidak sinkron
  • Analisis salah
  • Keputusan keliru
  • Risiko hukum dan keamanan

Elemen Pendukung DG

  • Master Data Management (MDM)
  • Standar data
  • Audit data
  • Dukungan manajemen puncak

3. Data Management

Terdiri dari beberapa komponen:

  • Dataset: data mentah awal
  • Log data: catatan aktivitas sistem
  • Data Warehouse: gudang data historis untuk analisis
  • Data Mart: subset data warehouse untuk unit tertentu

Fungsi Utama:

  • Analisis
  • Pelaporan
  • Pengambilan keputusan

4. It Architecture

Bab ini untuk memecahkan pertanyaan-pertanyaan sebagai berikut:

  • Bagaimana jika storage penuh?
  • Jika framework aplikasi tidak kompatibel dengan server?
  • Bagaimana support jangka panjang?

Konsep Penting:

  • Server
  • Storage
  • Network
  • Cloud vs Data Centers
  • Scalability

Teknologi Pendukung

  • Web Service / API: komunikasi antar sistem
  • SOA (Service Oriented Architecture): sistem modular
  • Timestamp: waktu akses data
  • Query: pengambilan data
  • Crawling: pengambilan data otomatis
  • IoT: contoh: CCTV
  • Enkripsi: pengamanan data

5. Cybersecurity

Pengertian

Cybersecurity adalah upaya untuk melindungi sistem informasi, jaringan, dan data dari ancaman digital seperti serangan, pencurian data, dan gangguan sistem.

Tujuan Utama

  • Menjaga kerahasiaan data (Confidentiality)
  • Menjaga keutuhan data (Integrity)
  • Menjamin ketersediaan sistem (Availability)

Risiko Keamanan

  • Data breach
  • Sistem down
  • Kehilangan data
  • Tidak bisa melakukan recovery

Pendukung Cybersecurity

  • Enkripsi data
  • Backup dan recovery
  • Audit keamanan
  • Framework seperti COBIT
  • Sertifikasi jaringan (contoh: CCNA)

6. Enterprise Systems

Level Sistem

  1. Operational Level
    • Transaction Processing System (TPS)
    • Mendukung transaksi harian
  2. Managerial Level
    • Management Information System (MIS)
    • Decision Support System (DSS)
  3. Strategic Level
    • Executive Support System (ESS)

Faktor Keberhasilan

  • Dukungan manajemen
  • Kualitas data
  • Pelatihan pengguna

Penyebab Kegagalan

  • Resistensi terhadap perubahan
  • Data tidak akurat
  • Manajemen proyek buruk
  • Kurangnya pelatihan

7. ERP, SCM, dan CRM

ERP (Enterprise Resource Planning)

Sistem terintegrasi untuk mengelola proses bisnis utama.

  • Keuangan
  • SDM
  • Produksi

Risiko ERP

  • Biaya tinggi
  • Keamanan
  • Kurangnya pelatihan SDM

SCM (Supply Chain Management)

Mengelola alur dari pemasok, produksi, distribusi, hingga pelanggan.

CRM (Customer Relationship Management)

Sistem untuk mengelola hubungan dan komunikasi dengan pelanggan.

Jenis CRM

  1. Operational CRM
  2. Analytical CRM
  3. Collaborative CRM
